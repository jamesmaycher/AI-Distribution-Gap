1. The Citizen Coder Moment

We are in the Citizen Coder moment. Non-technical users are building, deploying, and operating software at scale. The barriers to entry have collapsed. What started as an experiment with ChatGPT has become infrastructure.

This shift creates a new distribution problem: when non-experts build software, they rely entirely on AI guidance for architectural decisions, platform selection, and integration patterns. These recommendations become the effective distribution channel for infrastructure products.

Why This Matters

Traditional developer marketing targets engineers directly. But when a founder asks Claude "what's the best way to deploy a Python app," they are not consulting your marketing. They are consulting an AI that has never heard of your product, or worse, recommends a competitor.

3. The Agent Flywheel

A framework from Lavanya Shukla (Improbability.vc) explains why these patterns compound rather than correct over time.

When agents successfully complete tasks using a product, those execution patterns become training data for the next model generation. More successful completions strengthen model memory, which increases the probability of future selection, which generates more completions. It's a self-reinforcing loop.

THE AGENT FLYWHEEL
Agent uses product → Successful execution → Training data → Model memory → Higher default selection → ↻

6. The API Gap Problem

Many infrastructure products have gaps in API coverage that make them invisible to AI agents. If your product requires multi-step manual setup, AI will recommend something simpler.

The Opportunity

Products that expose complete, well-documented APIs have a 3-5x advantage in AI recommendation frequency. This is not a marginal improvement—it is a categorical difference in distribution.

Example: Testing API Coverage "I need to deploy a Python backend and database. Walk me through the steps using only API calls, no manual configuration."

Platforms that can answer this prompt with a complete, functional recipe win. Platforms that require manual steps lose.

11. The Opportunity Window

The agent distribution problem isn't permanent. AI infrastructure is in its first generation. Decisiveness now compounds over time.

Five Actions for Squarespace

1. Ship an MCP Server

Publish a Model Context Protocol server (like Wix, Railway, Supabase have done). Agents need programmatic access to CMS, inventory, and content creation.

2. Implement llms.txt

Publish squarespace.com/llms.txt with a comprehensive index of API capabilities. Make your platform discoverable by default.

3. Build a Claude Code / Cursor Plugin

Ship integration with the primary AI development interfaces (Claude Code, Cursor, Windsurf). These tools control the decision-making moment.

4. Optimize Docs for LLM Consumption

Restructure documentation with multi-language examples, clear parameter descriptions, and edge-case handling. LLMs parse structure differently than humans.

5. Expand the API Surface

Version 7.1's limited API is a distribution blocker. Programmatic site creation, domain management, and template selection must be API-first.

The window is still open. AI distribution is early enough that decisive action can change Squarespace's position within quarters, not years.

Vercel grew from invisible to $9.3B in 5 years. Railway went from 0 to $100M Series B largely through AI-driven growth. The distribution channel is young enough that the largest vendors today can be displaced if they don't adapt.

The Core Insight

Squarespace built for human designers. The next Squarespace will be built for human designers and AI agents. The question isn't whether to adapt—it's whether to adapt fast enough.
